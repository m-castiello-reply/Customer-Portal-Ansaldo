Contesto e Scope

Cosa copre la feature

In Scope

  • Integrazione documentale SF-MS/APIM/SPE/TC
  • Flussi upload, retrieval, view file
  • Gestione CommentSheet bidirezionale
  • Test E2E e supporto UAT stream integrazione
  • Preview strategy per formati supportati + fallback download

Out of Scope

  • Migrazioni massive documentali fuori flusso ordinario
  • Viewer custom e lifecycle avanzato non richiesto
  • Remediation infrastrutturali estese non pianificate
  • Gestione business completa ticketing IPS
  • Editing Office in-browser tramite viewer SharePoint nativo
  • Condivisione pubblica file con link diretti SPE/Graph

User Stories P2 e Edge Cases

Qualita operativa e readiness

US3 + US4

P2
  • View file tramite identificativo e permessi coerenti
  • Checklist prerequisiti e readiness go-live
  • Tracciamento test E2E e supporto UAT

Edge Cases critici

  • Mismtach identificativo file TC/SPE/SF
  • Timeout proxy on-prem o APIM
  • Permessi Graph incompleti
  • Metadata obbligatori mancanti
  • Accesso non autorizzato in view file
  • Formato non supportato: fallback obbligatorio a download
  • Token preview scaduto/alterato o riutilizzato
  • HTML EML/MSG non sanificato in preview

Requisiti Funzionali

Estratto FR-001...FR-018

Core Flow

  • FR-001/002: acquisizione e trasferimento file lungo la catena SF-APIM-Proxy-TC
  • FR-003/004: upload SPE, identificativo univoco e view nel portale
  • FR-005/006: supporto completo a CommentSheet e retrieval da SPE
  • FR-017: nessuna esposizione URL diretta SPE/Graph al browser

Controlli, Formati e Accesso

  • FR-007: esiti con codici errore + correlation id
  • FR-008: log tecnici su ogni hop
  • FR-009: integration test E2E obbligatori
  • FR-012: policy server-side preview vs download
  • FR-013/014/015: preview PDF/immagini/Office/EML-MSG via proxy
  • FR-016: ZIP/CAD con download controllato
  • FR-018: endpoint dedicati preview/download con auth Salesforce

NFR ed Entita Chiave

Sicurezza, osservabilita, manutenibilita

  • NFR-004: error taxonomy azionabile
  • NFR-005: documentazione e versioning interfacce
  • NFR-006: token firmati (HMAC), TTL breve, user-bound
  • NFR-007: hardening header HTTP preview/download
